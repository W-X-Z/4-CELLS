# 4 Cells — 세포 생태계 시뮬레이션 (MVP)

여러 종의 세포 군집이 자원과 환경을 주고받으며 자동으로 살아가는 생태계 시뮬레이션.
플레이어는 일정 시간마다 등장하는 3개의 선택지로 환경·세포·행동을 바꾸며 생태계의 균형을 유지한다.
모든 세포가 사라지면 게임 오버.

TypeScript + Vite + PixiJS 8 기반. 시뮬레이션 로직과 렌더링이 분리되어 있어 **그래픽 없이 헤드리스로 밸런스 테스트**가 가능하다.

---

## 실행 방법

```bash
npm install          # 의존성 설치
npm run dev          # 개발 서버 (브라우저에서 http://localhost:5173 열기)
npm run build        # 타입체크 + 프로덕션 빌드 (dist/)
npm run preview      # 빌드 결과 미리보기
npm run test         # 시뮬레이션 규칙 테스트 (Vitest)
npm run sim          # 그래픽 없이 시뮬레이션만 구동 (밸런스 확인)
```

`npm run sim -- <시드> <초> <전략>` 예: `npm run sim -- 12345 300 crisis`
(전략: `none` 무개입 / `random` 랜덤선택 / `crisis` 위기대응)

### 배포
`npm run build` 결과물(`dist/`)을 정적 웹 호스팅(Vercel, Netlify, GitHub Pages 등)에 올리면 된다.
설치 없이 URL 접속으로 플레이. `?seed=숫자`를 붙이면 동일한 전개를 재현할 수 있다.
(월드 크기가 화면 비율에 맞춰 생성되므로, 정확한 재현은 같은 화면 비율 기준.)

## 조작

- **선택지**: 카드를 클릭하면 효과가 적용되고 시뮬레이션이 재개된다.
- **배속**: 1× / 2× / 4× 버튼 또는 키보드 `1` `2` `4`.
- **일시정지**: ⏸ 버튼 또는 `Space`.
- 탭이 백그라운드로 가면 자동 일시정지.

## 코드 구조

```
src/
├─ core/          결정론 유틸 (시드 PRNG, Spatial Hash, 수학)
├─ simulation/    시뮬레이션 코어 (렌더링 의존성 없음)
│  ├─ World.ts        세계·틱 루프·점수/스냅샷
│  ├─ Environment.ts  전역 자원 풀
│  └─ systems/        Movement / Metabolism / Interaction / Reproduction / Death
├─ rules/         선택지 규칙 엔진
│  ├─ ConditionEvaluator.ts  조건 평가
│  ├─ EffectExecutor.ts      원자적 Effect 적용
│  └─ ChoiceSystem.ts        상황 가중치 기반 선택지 생성
├─ rendering/     PixiRenderer (시뮬레이션 상태를 읽어 그리기만)
├─ ui/            HUD 대시보드 / ChoicePanel / GameOverPanel / FeedbackLayer
├─ data/          species / environments / choices (JSON형 데이터 + Zod 검증)
└─ game/Game.ts   고정 타임스텝 루프 오케스트레이션 (헤드리스 겸용)
```

핵심 원칙: **시뮬레이션은 렌더링을 모른다.** `World`는 순수 로직이라 Node에서 그대로 돌려 테스트한다.

## 게임 설계 메모

### 자원 순환 루프 (연쇄 붕괴의 뼈대)
```
광합성 : CO₂ → O₂                      (에너지 획득 / 유일한 O₂ 생산자)
호흡   : O₂ → CO₂  (살아있는 모든 세포)  기초대사에 O₂를 쓴다. 부족하면 질식.
소비   : 광합성 세포 포식                (초식 — 광합성 개체수 조절)
포식   : 소비 세포 포식                  (육식)
분해   : 유기물(사체) 섭취 + 독성 정화    (에너지 획득)
사망   : 유기물 + 독성을 풀로 환원
```
- **빛은 제거**됐다(항상 존재하는 배경 조건). 광합성의 유일한 원료 제약은 CO₂.
- **O₂는 생존 자원**: 모든 세포가 호흡으로 O₂를 소비하고 CO₂를 배출한다. O₂가 마르면 부족분에
  비례해 에너지를 잃는다(`suffocationPenalty`).
- 먹이사슬: 생산자(광합성) → 초식(소비) → 육식(포식), 분해자는 사체·독성을 순환에 되돌린다.
→ 소비 세포가 광합성을 과도하게 뜯으면 **O₂ 생산 붕괴 → 전 세포 질식**. 광합성 단일종화도 자원
  고갈로 무너진다. 한 고리를 건드리면 도미노가 넘어지도록 **의도적으로** 엮여 있다.

### scavenge (기회적 정화)
분해 세포는 유기물로 살아가되(주 에너지원), 독성은 `scavenge`로 있으면 정화하고 없으면 굶지 않는다.
대사 satisfaction은 필수 `intake`의 최솟값이라, 독성을 필수로 두면 독성이 0일 때 굶어 죽는 문제를
피하기 위한 별도 채널이다.

### 결정론(시드 재현)
- `Math.random()` 미사용. 모든 무작위는 시드 기반 `Rng`(mulberry32)에서 나온다.
- 고정 타임스텝(`1/simRate`) + 안정적인 세포 순회 순서.
- 같은 시드 → 같은 결과 (테스트로 검증됨).

### 선택지 = 상황 가중치
완전 무작위가 아니라, 위기 상황(예: 산소 부족)에서 대응 선택지의 등장 확률이 올라간다.
"내 결정으로 대응했다"는 감각을 위해서다. 선택지는 코드가 아니라 `data/choices.ts`의
**Effect 조합 + 조건**으로 정의 — 새 선택지는 데이터만 추가하면 된다.

### UI (모바일 우선)
- 상단: 플레이 시간 + 배속/일시정지. 그 아래: 환경 자원(산소·CO₂·유기물·열·독성)을 **숫자 카드**로
  표시(위험 수준이면 붉게 강조). 하단: 종별 개체수 칩.
- **세포 칩을 탭하면 일시정지 + 도움말 모달** — 종의 역할, 현재 대사(호흡·정화 포함), 적용된 진화(선택 이력).
- 선택 적용 시 우하단에 효과 토스트로 무엇이 바뀌었는지 즉시 표시.
- **시각 효과**: 포식이 일어나면 그 위치에 `eat!`, 광합성이 활발하면 `O₂` 텍스트가 떠오른다.
- 세포는 에너지가 낮으면 흐려지고, 상호작용 시 반짝인다.
- 세포는 **색 + 모양**으로 구분(색약 대응): 광합성 ●원 / 소비 ◆다이아 / 포식 ▲삼각 / 분해 ○링.

### 점수
생존 시간 × (1 + 생물다양성) + 바이오매스 + 개체수. "얼마나 오래, 다양하게 번영시켰나"를 측정.

## 밸런싱 노트 (튜닝 포인트)

숫자는 전부 `src/data/`에 모여 있어 코드 수정 없이 조정 가능하다.

- **선택 주기**: `ChoiceSystem`의 `interval`(기본 18초). 게임 체감을 좌우하는 핵심 값 — 실기기 플레이로 조정 권장.
- **초기 밸런스**: 무개입 시 대체로 ~200초에 붕괴(개입이 필요한 생태계). 개입(crisis)하면 유지 가능.
  시드별 편차가 있다(카오스적 진동) — `npm run sim -- <시드> <초> none`으로 빠르게 확인.
- **핵심 튜닝 레버**:
  - `respirationRate` / `suffocationPenalty`(`environments.ts`): O₂ 압력과 질식 강도.
  - `respirationCo2Ratio`: 호흡이 CO₂를 얼마나 되돌리는지 — 광합성 개체수를 좌우.
  - 소비 세포 `attackEnergy`/`divideEnergy` + `VISION`(`Movement.ts`): 초식 압력과 저밀도 먹이 피난처.
    시야가 클수록 초식자가 먹이를 잘 찾아 과방목→붕괴가 쉬워진다.
- 자원 풀은 유한한 저수지로 `displayCaps` 상한에서 clamp된다(폭주 방지).

## 성능

- 세포 탐색은 **Spatial Hash**(모든 쌍 비교 O(n²) 미사용). 충돌은 거리 기반만(범용 물리 엔진 없음).
- 기기 성능에 따라 최대 세포 수/해상도/효과 자동 조절(`core/device.ts`): 모바일 ~1,000 / PC ~2,500.
- 렌더는 스프라이트 풀링. (대규모 확장 시 `ParticleContainer`로 업그레이드 여지 있음.)
